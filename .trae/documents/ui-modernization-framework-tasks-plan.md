# UI 现代化改进计划（主框架 + 任务页）

## Summary
- 目标：在不破坏主流程的前提下，对当前 UI 做“极简商务”风格升级，重点覆盖主框架与任务页。
- 成功标准：统一视觉规范（字体、间距、组件层级、状态反馈）、解决裁切与拥挤、增强轻交互（悬停/选中/空状态/批量操作反馈）、保持双语与全局缩放一致性。
- 范围：`MainWindow`、`TasksView`、`TaskAnalysisWindow` 及对应 ViewModel/UI 资源；不改核心同步业务逻辑。

## Current State Analysis
- 全局主题资源集中在 `App.xaml`，已有自定义色板与 `SideNavButtonStyle`，但页面级样式复用不足。
- `MainWindow.xaml` 已接入全局字体与缩放、支持左侧导航，但导航与主内容视觉分层仍偏“工具原型”，现代感不足。
- `TasksView.xaml` 功能密度高（多选/统一按钮/状态图标/三点菜单），信息层级与留白尚不统一，卡片与操作区视觉焦点不够明确。
- `TaskAnalysisWindow.xaml` 为功能可用形态，数据可读性（列优先级、密度、选中态、状态突出）还有优化空间。
- `SettingsView.xaml` 已有语言/字体/缩放配置，可作为视觉规范入口，但控件节奏（行高、分组、强调）可再统一。
- 资源字典有中英文 `Strings.zh-CN.xaml` 与 `Strings.en-US.xaml`，说明文案国际化基础已具备。

## Proposed Changes

### 1) 建立“统一视觉基线”资源（主框架与任务页共用）
- 文件：`App.xaml`
- What：
  - 新增统一间距、圆角、阴影、边框、标题字号、正文字号资源（例如 `Space.*`, `Radius.*`, `Elevation.*`, `Typography.*`）。
  - 补充任务页专用语义色（成功/警告/中性/选中背景）与状态图标色。
  - 抽取可复用样式：统一按钮样式（Primary/Secondary/Ghost）、列表行 hover/selected 样式、信息标签样式。
- Why：减少“页面各自为政”导致的不一致，先统一规则再做页面细化。
- How：保持 MaterialDesign 基础样式 `BasedOn`，以资源覆盖方式增强，不直接硬编码颜色与尺寸。

### 2) 主框架现代化（导航、页头、内容区层级）
- 文件：`MainWindow.xaml`
- What：
  - 优化左侧导航块：图标与文本对齐、选中态更清晰（背景+细边强调）、hover 态更轻。
  - 调整侧栏与右侧内容的层级关系：弱化过重阴影，强化卡片分组与标题节奏。
  - 统一页面容器边距与可读宽度，避免缩放后“挤压感”。
- Why：主框架是全局视觉锚点，先稳住整体“现代商务”观感。
- How：基于现有 `SideNavButtonStyle` 与主容器 Border/Shadow 进行参数化调整，不改导航逻辑。

### 3) 任务页信息架构优化（一致性优先）
- 文件：`UI/Views/TasksView.xaml`
- What：
  - 顶部统一按钮区重排：左侧批量状态信息 + 中部筛选/提示 + 右侧分析/执行/同步主操作，建立视觉主次。
  - 任务列表项重构为“信息区 + 状态区 + 操作区”三段，统一行高与内边距。
  - 分析状态图标（? / ✅）强化语义颜色与可点击反馈；复选框、任务标题、状态标签对齐。
  - 三点菜单入口做统一尺寸与触达区，编辑/删除动作增加危险级视觉区分。
  - 空状态、加载状态、分析进行中状态文案与图形统一。
- Why：当前功能完整但层级密集，用户扫描成本高；本项直接提升可读性与效率。
- How：仅重排 UI 结构与样式，不改已有批量分析/执行/同步流程。

### 4) 分析窗口可读性与操作体验优化
- 文件：`UI/Views/TaskAnalysisWindow.xaml`, `UI/ViewModels/TaskAnalysisViewModel.cs`
- What：
  - DataGrid 列优先级重排：同步勾选、路径、动作方向、原因置前；大小/时间列分组显示。
  - 增强“可执行项在前”的视觉强化（行底色或标签），并保留手动修改。
  - 保存按钮与执行按钮形成明确主次（保存为次主操作，执行为主操作）。
  - 增加轻交互反馈：保存成功提示、未保存修改提示（轻量，不改主流程）。
- Why：分析窗口是“决策点”，需要更高的信息可读性与操作确定性。
- How：优先通过 XAML 样式与绑定状态实现；ViewModel 仅补充必要状态字段与提示。

### 5) 双语与显示一致性收口
- 文件：`UI/Localization/Strings.zh-CN.xaml`, `UI/Localization/Strings.en-US.xaml`, `UI/Localization/LocalizationService.cs`, `UI/Views/TasksView.xaml`, `UI/Views/TaskAnalysisWindow.xaml`
- What：
  - 补齐本轮新增/优化控件文案的中英文资源键。
  - 检查缩放与字体切换在主框架、任务页、分析窗中的一致行为（不出现截断/错位）。
- Why：你明确只需中英双语，并强调一致性优先；这一步用于防回归。
- How：新增资源键并替换硬编码文案；不引入第三语言。

### 6) 文档同步更新（结构与能力）
- 文件：`CLAUDE.md`
- What：同步“主框架+任务页现代化”的实际实现点与资源组织（如新增样式键、页面结构变更）。
- Why：文档是协作入口，需保证 co-work 同步。
- How：仅更新对应章节，不扩写无关内容。

## Assumptions & Decisions
- 已确认风格：极简商务。
- 已确认优先范围：主框架 + 任务页。
- 已确认改动边界：轻交互增强，不改主业务流程。
- 已确认验收偏好：一致性优先。
- 不引入新 UI 框架；继续基于 WPF + MaterialDesignThemes。
- 仅支持中文/英文，不扩展多语种。

## Verification Steps
- 视觉一致性检查：
  - 导航、任务页、分析窗是否使用统一字号层级与间距节奏。
  - 主按钮、次按钮、危险操作样式是否一致。
- 功能回归检查：
  - 任务页多选、统一按钮（分析/执行/同步）、三点菜单行为不变。
  - 分析状态图标逻辑（? -> ✅，点击✅打开分析窗）不变。
  - 分析窗“保存/执行”逻辑不回退。
- 显示适配检查：
  - 缩放 `0.8x / 1.0x / 1.3x / 1.5x / 2.0x` 下无裁切与严重错位。
  - 中英切换后文案无溢出/遮挡，关键页面布局稳定。
- 构建检查：
  - `dotnet build -c Release` 通过。
  - 发布包可正常启动并验证以上视觉与交互项。
